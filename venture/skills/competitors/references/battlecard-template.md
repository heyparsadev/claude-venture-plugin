# Battlecard Template

One page per key competitor. Audience: whoever sells/pitches against them (founder, sales, or the consulting client). Keep it brutally practical — this gets read 30 seconds before a call.

```markdown
# Battlecard: us vs <Competitor>
<date> · threat: <high/med/low>

## Them in one line
<what they are and who they're best for — honest, no strawman>

## Quick compare
| | Us | Them |
|---|---|---|
| Best for | | |
| Pricing | | |
| Key strength | | |
| Key weakness | | |
| Deal-breaker gap | | |

## When we win
<2–3 scenarios/customer profiles where we're genuinely the better choice — and the proof points to use>

## When we lose (be honest)
<scenarios where they're the better fit — qualify these out early instead of fighting>

## Their pitch against us — and the counter
- **They'll say:** "<likely objection>" → **Counter:** <response grounded in facts>
- **They'll say:** "<...>" → **Counter:** <...>

## Landmines to plant (fair, factual)
<questions a buyer should ask them that expose their real weaknesses —
e.g. "ask them what happens to your data if you cancel" — only verifiable ones>

## Objection handling
| Prospect says | Respond |
|---|---|
| "We already use <competitor>" | <switch-cost-aware response> |
| "<competitor> is cheaper" | <value reframe or honest concession> |

## Proof points
<our evidence: metrics, testimonials, case studies — only real ones; mark TBD if none yet>
```
