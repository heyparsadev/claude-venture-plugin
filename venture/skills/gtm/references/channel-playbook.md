# Channel Playbook — Bullseye Selection

## Channel menu (brainstorm across ALL before narrowing)

| Channel | Works best when | Watch out |
|---|---|---|
| Founder-led outbound (DM/email/calls) | B2B, price >$1k/yr, identifiable ICP | doesn't scale past founder; that's fine early |
| SEO / content | problem is actively searched; 6–12mo patience | slow signal; check search volume first |
| Paid search | high-intent queries exist; price supports CAC | bid costs vs LTV; needs landing page + tracking |
| Paid social (IG/TikTok/local equivalents) | visual/consumer; impulse-friendly price | creative treadmill; weak for niche B2B |
| Marketplaces & app stores | category browsed there (apps, plugins, templates) | platform tax + ranking dependence |
| Partnerships / resellers | partner already owns your ICP's trust | slow to start; revenue share |
| Communities & social proof (forums, Telegram/LinkedIn groups, local associations) | tight-knit ICP that talks to itself | give-before-ask; ban risk for spam |
| Events / webinars / local meetups | high-touch B2B; local-market trust building | time-expensive per lead |
| Referral / viral loops | product is visible or collaborative by nature | needs existing happy users — not a cold-start channel |
| PR / launch platforms (Product Hunt, local tech media) | novelty story; one-time spike | spike ≠ channel; harvest emails from it |

Local-market note: map to local equivalents (Iran: Telegram channels/groups, Instagram, Cafe Bazaar/Myket, اتحادیه‌ها و انجمن‌های صنفی, local tech media like Digiato/Zoomit).

## Scoring (fill for every plausible channel)

Pick **1–2 core + 1 experimental**. Spreading across 5 channels = learning nothing about any.

## Experiment card (one per chosen channel)

```markdown
### Channel: <name>
- Hypothesis: <ICP X is reachable here at CAC ≤ Y>
- First experiment: <concrete: "50 personalized DMs to clinic owners in Tehran via Instagram, message variant A/B">
- Budget cap: <money + hours>
- Success metric: <e.g. ≥10% reply, ≥2 demos booked>
- Kill criterion: <e.g. <2% reply after 100 sends → stop, write down why>

```

## Economics sanity check

CAC ceiling ≈ first-year gross profit per customer (early stage). Channel can't plausibly hit that → don't pick it, note why. Mark every CAC figure as estimate (confidence) until real data replaces it.
