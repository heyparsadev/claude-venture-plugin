# Sean Ellis PMF Survey — Template & Scoring

## The core question

> How would you feel if you could no longer use **<product>**?
> 1. Very disappointed
> 2. Somewhat disappointed
> 3. Not disappointed (it isn't really that useful)
> 4. N/A — I no longer use it

**Benchmark: ≥40% "very disappointed" = strong PMF signal.** 25–40% = close, mine the "very disappointed" segment for direction. <25% = fit not found yet.

## Full survey (customize per product, translate to the customers' language)

1. The core question above.
2. What type of people do you think would most benefit from <product>? *(reveals the real ICP in users' own words)*
3. What is the main benefit you receive from <product>? *(the actual value prop — often ≠ the founder's)*
4. How can we improve <product> for you?
5. (optional) What would you likely use as an alternative if <product> were no longer available?

Persian rendering of the core question (keep scale order identical):

> اگر دیگر نتوانید از **<محصول>** استفاده کنید چه احساسی خواهید داشت؟
> ۱. خیلی ناراحت می‌شوم
> ۲. کمی ناراحت می‌شوم
> ۳. ناراحت نمی‌شوم (واقعاً آن‌قدر مفید نیست)
> ۴. دیگر استفاده نمی‌کنم

## Sampling rules (violating these voids the result)

- Ask only people who **experienced the core value**: used the product ≥2 times, within the last 2 weeks.
- Minimum **n ≈ 30–40** responses for a readable signal; below that, treat as anecdote.
- Don't survey only fans (e.g. a Telegram superuser group) — sample across the active base.
- Send from a person, short intro, 2 minutes max, no incentives that attract noise.

## Reading the result — the segment cut is the whole point

Cut "very disappointed" % **by segment** (role, company size, use case, acquisition channel):

- Overall 25% but one segment at 55% → you have PMF *in that segment* — narrow focus to it (this is the classic Superhuman playbook).
- Use Q2/Q3 answers from the "very disappointed" group verbatim in positioning (`/venture:gtm` consumes these).
- "Somewhat disappointed" group's Q4 answers = the roadmap to convert them; ignore improvement requests from "not disappointed" (they were never the market).
