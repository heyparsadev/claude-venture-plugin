# Lean Canvas Template (Ash Maurya)

One page, nine boxes. Each cell cites where it came from (`[discovery]`, `[market]`, `[competitors]`, `[gtm]`, `[assumption]`). TBD beats fabrication.

```markdown
# Lean Canvas — <Venture>
<date> · iteration <N>

## 1. Problem (top 3)
<from discovery synthesis pains table — with frequency×severity rank>
Existing alternatives: <what they use today — from discovery/competitors>

## 2. Customer segments
<target + early adopters specifically: who buys FIRST [discovery]>

## 3. Unique value proposition
<single clear message — reuse positioning statement if GTM ran [gtm]>
High-level concept: <"X for Y" if helpful>

## 4. Solution (top 3 features only)
<mapped 1:1 to the problems above — nothing else belongs here>

## 5. Channels
<from GTM channel selection; else hypothesis [assumption]>

## 6. Revenue streams
<pricing model + price point + rationale [gtm/competitors]>

## 7. Cost structure
<fixed (team, infra, rent) + variable per customer (COGS, support, payment fees)>

## 8. Key metrics
<the 3–5 numbers that show the engine works — match the business type
(e.g. SaaS: signups→activated→retained→paying; marketplace: liquidity, repeat rate)>

## 9. Unfair advantage
<what can't be easily copied or bought: distribution lock-in, proprietary data, community,
regulatory license, founder insight. "None yet" is an honest and common answer — write it
and note the plan to build one.>

---
**Riskiest cell right now:** <which box, why, and the cheapest test for it>
```

Order of filling (matters): 1 → 2 → 3 → 9 → 4 → 5 → 6 → 7 → 8. Problem before solution, always.
