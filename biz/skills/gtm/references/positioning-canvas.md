# Positioning Canvas (April Dunford method)

```markdown
# Positioning — <Venture>
<date>

## 1. Competitive alternatives
<What the target customer actually does today without us — including manual/status-quo options.
Ranked by how often discovery/competitor research saw them. Cite where each came from.>

## 2. Unique attributes
<Capabilities/features/traits we have that the alternatives genuinely lack.
Each must survive the test: "can alternative X claim this too?" If yes, cut it.>

## 3. Value (so what?)
<For each attribute: the customer outcome it enables. Attribute → value chain, e.g.
"automated SMS reminders → 30–50% fewer no-shows → recovered revenue without new patients".>

## 4. Who cares most (target segment)
<The customer profile for whom that value is burning: firmographics/demographics + situational
triggers ("just lost a receptionist", "opening second location"). This sharpens the ICP.>

## 5. Market category & frame
- **Category chosen:** <existing category | subcategory | new category> — <rationale: comprehension vs differentiation trade-off>
- **Framing:** "<category> for <segment>" or "<familiar thing> but <key difference>"

## Positioning statement
For **<segment>** who **<trigger/pain>**, **<product>** is a **<category>** that **<key value>**.
Unlike **<main alternative>**, it **<primary differentiator>**.

## Evidence base
<which findings from competitors/customers/pmf modules support each section — with file refs>
```

Test before shipping: read the statement to the ICP description — would that person interrupt you to ask "how?" If it reads like it could describe three competitors, return to section 2.
