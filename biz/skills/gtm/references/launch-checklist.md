# Launch Plan Template

Calibrate to reality: "launch" for a B2B niche tool = 20 warm emails + 3 community posts. For consumer = coordinated spike. Both are launches; size the checklist accordingly, assign an owner and date to every item.

```markdown
# Launch Plan — <Venture>
<date> · launch type: <soft beta | public launch | new-market entry> · target date: <date>

## Goals (pick 1 primary)
- Primary: <e.g. 100 signups week 1 | 10 demo calls | 30 beta users giving feedback>
- Secondary: <email list, social follows, press mention>

## Pre-launch (T−4 to T−1 weeks)
- [ ] Positioning & messaging finalized (from messaging house) — <owner/date>
- [ ] Landing page live with analytics + signup path tested end-to-end
- [ ] Beta/waitlist warm-up: <how many contacts, where from>
- [ ] Assets: demo video/screenshots, launch post drafts per channel, FAQ
- [ ] Onboarding ready for strangers (someone outside the team completed it unaided)
- [ ] Support path: who answers, where (email/Telegram/chat), response SLA
- [ ] Pricing & payment tested (real transaction + refund)

## Launch window (day-by-day)
| Day | Action | Channel | Owner |
|---|---|---|---|
| D0 | <announce to warmest audience first> | | |
| D1 | <community/platform posts> | | |
| D2–6 | <follow-ups, engage every comment/reply, daily metric check> | | |

## Post-launch (the part that matters)
- [ ] Week 2: personally contact first ~20 users — what almost stopped them?
- [ ] Week 2–4: double down on the one channel that produced signal; kill the silent ones
- [ ] Week 4: activation/retention snapshot → feeds `/biz:pmf`
- [ ] Week 4: retro — what to repeat next launch

## Risk prep
- <top 2 launch risks (e.g. onboarding breaks under load, crickets) + mitigation for each>
- Crickets plan: if <metric> < <floor> by D3 → <pre-decided response: revise hook, second wave, direct outreach>
```
